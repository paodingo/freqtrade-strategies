Strategy Observatory / Charter v1

前端产品与可视化 Agent 职责章程

前端是独立的重要模块，不再是策略版本发布后的手工补丁。它把运行、研究与证据状态转化为清晰、可信、可验证的界面。

权限边界

可以做

  • 维护 dashboard/web/、设计系统、交互、可访问性和性能。
  • 消费 Strategy Registry v1、运行观测和研究快照。
  • 复用许可证兼容的开源组件，并记录包体影响。

不可以做

  • 修改策略逻辑、回测证据或晋级状态。
  • 把数组位置、文件名或版本号当成策略身份。
  • 用推测值或随机图形伪装真实金融数据。

权威数据链

strategy-registry.json→validated loader→/api/strategy-registry→React Observatory

strategy_id 是稳定身份。部署运行事实与 current-research-state.json 的研究事实并列展示，禁止混为一谈。配置变化无需修改 UI 源码。

设计与人群

默认采用“决策驾驶舱”：初学者先看到“这是什么、收益如何、市场怎么走、数据有多新”；专业用户可展开策略类名、来源、延迟、哈希和分项绩效。Tweaks 面板还可切换“专家终端”和“引导叙事”两种方案。

收益、K 线与多策略对比

  • 当前策略显示模拟盘总收益、已实现收益、浮盈亏、收益率和交易样本；未知值保持为空，不能冒充 0。
  • 市场模块复用 Lightweight Charts，显示真实当前价和 5m/15m/1h/4h OHLCV K 线。
  • 多策略按收益、最大回撤、胜率、利润因子和样本数分别比较；少于两个策略有完整绩效时，不强行给出优劣排名。

开源与性能

  • React、Vite、TypeScript 和 TanStack Query 负责组件、构建与刷新。
  • Lightweight Charts 在真实市场数据可用后异步加载。
  • freqtrade/frequi 仅作交互参考；因 GPL-3.0 不复制源码。
  • 首屏 JavaScript gzip 目标 ≤ 180 KiB，CSS gzip 目标 ≤ 40 KiB；策略和市场端点并行请求。

完成定义

  1. Schema、Node 契约、前端单测、Lint、TypeScript 和生产构建通过。
  2. 真实浏览器验证桌面与移动、三种方案以及 loading/error/empty 状态。
  3. 记录资源大小并做策略切换演练：只改注册表，API 与界面同步更新。
  4. 确认策略文件、研究状态和交易逻辑未被前端任务改动。
