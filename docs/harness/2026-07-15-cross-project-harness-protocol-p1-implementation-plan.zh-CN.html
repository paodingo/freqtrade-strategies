01 · OUTCOME

P1 的确定交付与停止边界

交付：Protocol v0.1 schema、manifest、五个 golden fixtures、三组 isolated tests、exact-path guard。Schema 验证形状，测试验证跨工件语义。

Draft 2020-125 fixtures16 testsexit 0/1/2exact paths

不授权：CLI、package、plugin、executor、Adapter、Role Pack、state backend、database、network、model provider、production integration，也不修改 ChinaSectorRadar 或 rehab-intervention。
  • 每个 Task 前必须 clean；每次只 stage 当前 Task 的精确路径。
  • 只使用 synthetic、non-secret 数据；Core JSON 不出现项目名或领域字面量。
  • 0/1/2 固定映射为 passed/blocked/error。
  • business_readiness 与 harness_completion 始终独立。
当前 baseline condition：2026-07-15 的完整 clean_worktree_portable 用时约 793 秒，fixture pack 通过且未修改 versioned worktree，但因本地 runtime 中已有 418 个 2026-07-14 创建的 __pycache__/*.pyc，产生两个同源的新失败。P1 计划不授权删除它们；实施前必须使用干净 hydrated runtime，或另行批准精确缓存清理后重跑。不得把该状态写成 baseline 通过，也不得隐藏其余已识别的历史债务。

05 · TASK 3

固定四类 failure semantics

Fixture必须证明结果
governed-blockattempted path 同时位于 allowed 和 blocked；blocked 优先blocked / 1 / path_blocked
tool-error环境或工具不可用不是治理违规error / 2 / environment_unavailable
authority-mismatch绑定 fingerprint 与当前 authority 不同，必须 fail closedblocked / 1 / authority_mismatch
known-baseline-debtHarness 正确完成并保留债务；业务仍 blockedpassed / 0 / known_baseline_debt_preserved

Conformance tests 验证所有 fixture documents、expected outcomes、exit mapping、path precedence、tool error、authority mismatch，以及 debt/readiness/completion 分离。此时合计 13 tests。
