Cross-Project Harness Protocol P2 只读映射审计

日期：2026-07-15 · 状态：mapping_only_ready · 协议基线：harness-protocol-v0.1

结论

P2 可以进入实施计划，但只授权项目事实到 Protocol contract 的只读映射。三个项目都具备可用来源，却没有任何一个能直接覆盖全部 11 个 contract。

成功标准：不是把所有项目强行变成同一种 Harness，而是证明项目本地 authority、禁止项、known debt 和 evidence truth 不会被共享 Protocol 弱化。

关键发现

freqtrade-strategies

治理、fingerprint approval、budget、role boundary 和 append-only state 最成熟；但 state snapshot 与后续 human decision 时间不同，不能拼成一个虚假的新鲜快照，EvidenceBundle 仍是分布式来源。

ChinaSectorRadar

P0 的 H0/planned 描述已经过时：当前是 H1，manifest、readiness 和 evidence package 已实现。H1.1 independent review 已通过，但 readiness 仍为 blocked/1 并保留四个 H2 blocker；candidate-time evidence 的 review pending 与随后 reviewer pass 必须按 lifecycle 同时保留。下一动作仅可请求 H2 计划审批。

rehab-intervention

本地 runner 的 task、budget、state、retry 最完整；但 state 是 ignored local artifact，不是 portable authority。项目没有正式 RoleContract 和 fingerprint-bound ApprovalRecord。

不变量与边界

  1. 项目禁止项只能保持或收紧。
  2. gap 不能生成伪造 contract。
  3. blocked 与 error 必须分离。
  4. Harness completion 与 business readiness 必须分离。
  5. source commit、fingerprint、timestamp 必须绑定。
  6. P2 不 import 或执行项目 Runtime。

下一道门：单独审批 P2 implementation plan。此前不创建 mapping descriptor，不修改兄弟仓库，也不选择 P3 pilot。
