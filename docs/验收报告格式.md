请将验收报告改成正式决策报告格式，不要只输出简单 status: pass。

报告必须包含以下部分：

一、最终结论

输出以下四种状态之一：

1. fail：不通过，不能进入下一阶段。
2. conditional_pass：有条件通过，需要修复关键问题后再进入下一阶段。
3. paper_ready：允许进入 dry-run / 模拟盘验证。
4. live_ready：允许小资金实盘验证。

当前阶段默认只能最高评为 paper_ready，不允许直接评为 live_ready。

二、硬性验收表

逐项输出：

* 指标名称；
* 目标值；
* 实际值；
* pass / fail；
* 证据文件；
* 简短诊断。

第二阶段硬性指标如下：

1. 70天标准成本核心组合净收益 >= 5%。
2. 最近30天标准成本核心组合净收益 >= 3%。
3. 最近30天核心组合交易数 >= 20。
4. 最大回撤 <= 8%。
5. 最大单笔盈利贡献 <= 30%。
6. 滑点 x2 + taker 保守成本下，70天核心组合仍为正收益。
7. 趋势回踩未修复前必须保持 research_only。
8. OI 必须保持 optional research layer，不得作为 baseline 必需条件。
9. 不允许通过提高杠杆、提高单笔风险、删除成本、未来函数或数据泄露来达标。

三、强通过指标

强通过不是硬性要求，但需要输出：

1. 70天标准成本核心组合净收益 >= 7%。
2. 最近30天标准成本核心组合净收益 >= 5%。
3. 保守成本下70天净收益 >= 3%。
4. Profit Factor >= 1.5。
5. Payoff Ratio 接近或高于 1。
6. 滚动30天窗口大部分为正。
7. 没有多个滚动窗口出现最大单笔盈利贡献 >30%。

四、策略臂诊断

分别输出：

1. 震荡均值回归：是否仍是核心贡献来源，成本是否过高。
2. 波动压缩突破：交易数是否仍然过少，是否存在单笔依赖。
3. 趋势回踩：是否继续 research_only，是否禁止进入核心组合。
4. OI enhanced：相对 baseline_no_oi 是增益还是减益。

五、成本压力诊断

必须比较：

1. 标准成本；
2. 滑点 x1.5；
3. 滑点 x2 + taker。

输出成本压力下收益衰减比例。如果保守成本下收益明显变薄，需要标记为 cost_sensitive。

六、滚动窗口稳定性

必须输出：

1. rolling 14d；
2. rolling 30d；
3. 每个窗口净收益；
4. 每个窗口最大回撤；
5. 每个窗口最大单笔盈利贡献；
6. 是否存在短窗口单笔依赖。

七、最终决策

根据以上结果输出：

* 是否允许进入模拟盘；
* 是否禁止进入实盘；
* 下一阶段必须修复的问题；
* 推荐的下一阶段任务清单。

不要因为顶层 status 为 pass 就直接给出 live_ready。只要没有经过 dry-run / paper trading，最终状态最多只能是 paper_ready。
