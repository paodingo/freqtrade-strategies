OVERVIEW

目标、架构与技术栈

Agent worker 要求：逐 Task 实施本计划，并使用 checkbox（- [ ]）追踪步骤。

目标：建立受治理、事件触发的发现层：Researcher 提出新策略族，独立 Research Critic 发起质疑，人工最多选择一个方向，仅由既有 Research Director 将其转换为尚未执行的正式 proposal。

架构：Agent worker 与 provider 无关，通过已提交的 prompt packet 生成 schema-bound JSON；仓库不调用 model API，也不读取 model credential。确定性 Python 负责来源验证、固定范围、排序、不可变 fingerprint、状态迁移、人工 approval 绑定、registry 写入、审计输出与 Director handoff。第一次 acceptance run 停在 Candidate 创建或 Campaign 执行之前。

技术栈：来自 .venv-freqtrade/Scripts/python.exe 的 Python 3.12、standard library、jsonschema 4.26.0、SQLite Director Registry、仓库 simple-YAML loader、unittest、PowerShell readiness checks 与 Node.js guard scripts。

GLOBAL

  • 只能使用 D:\code\freqtrade-strategies-clean\.venv-freqtrade\Scripts\python.exe；不得使用 python、py 或 PATH discovery。
