# 策略说明书

## 当前目标

当前线上不是追求“版本越多越好”，而是做一组干净的双 bot 对比：

- **V6.3**：当前稳定基线，在 V6.2 信号基础上按账户最大亏损预算决定加仓额度。
- **V6.5**：当前进攻挑战者，面向震荡期小波段，沿用账户风险预算，但重新开启并放宽震荡入场。
- **V6.6**：新候选策略，面向“趋势后高波动震荡”，只在 24h/48h 箱体边缘交易，并在箱体中部保持克制。

我们要观察的是：V6.5 能否在震荡期捕捉更多 15m 小波段，同时仍然受账户风险预算、止损/强平距离、波动和余额约束。
V6.6 的观察重点不同：它是否能牺牲一部分交易频率，换来更好的入场位置、较小回撤和更快的错误承认。

## 核心机制

当前版本都基于 4h 判断市场状态。V6.3 在 1h 上找趋势入场，V6.5 在 15m 上找更快的趋势和震荡短线入场，V6.6 在 V6.5 基础上额外判断 24h/48h 箱体位置。

- 趋势判断：EMA、ADX、DI、ATR 等指标确认方向和趋势强度。
- 入场确认：在趋势方向上等回调、反弹或突破。
- 出场控制：ROI、固定止损、自定义价格滑点和超时保护共同约束。
- 风险控制：每个 bot 独立 dry-run 数据库，当前采用单 BTC 仓、`max_open_trades=1`。

## V6.3 与 V6.5 的区别

| 项目 | V6.3 稳定基线 | V6.5 震荡进攻挑战者 |
| --- | --- | --- |
| 趋势入场 | 保留 | 保留 |
| 震荡入场 | 关闭 | 开启并放宽 |
| 保护机制 | Cooldown、StoplossGuard、MaxDrawdown | Cooldown、StoplossGuard、MaxDrawdown |
| 主交易周期 | 1h | 15m |
| 首笔仓位 | 1,500 USDT | 3,000 USDT |
| 单次加仓目标 | 1,000 USDT | 1,500 USDT |
| 最大总仓位 | 3,500 USDT | 6,500 USDT |
| 加仓盈利门槛 | 每次 1.2% 起 | 每次 0.4% 起 |
| 加仓冷却 | 60 分钟 | 15 分钟 |
| 账户最大亏损预算 | 1.5% | 3.5% |
| ATR 波动上限 | 3.0% | 5.0% |
| 额外门槛 | 止损/强平距离、波动、冷却、可用余额 | 同 V6.3，但阈值更进攻 |

V6.5 不是“把钱用满”的版本。它仍然按账户最大亏损预算反推可加仓额度：如果止损距离太近、强平距离太近、ATR 波动过高、冷却时间未到、可用余额不足，或者账户风险预算已经用完，都不会加仓。

## V6.6 的操盘逻辑

V6.6 不追求比 V6.5 更密集交易，而是把震荡行情先拆成“边缘、中心、突破”三种位置：

- 上沿附近：只有 24h/48h 箱体位置都偏高，并且布林位置与 RSI 同时过热，才允许 `v66_ranging_short_edge`。
- 下沿附近：只有 24h/48h 箱体位置都偏低，并且布林位置与 RSI 同时过冷，才允许 `v66_ranging_long_edge`。
- 箱体中部：默认不新开震荡仓，避免在盈亏比最差的位置猜方向。
- 趋势失效：如果旧趋势单遇到 4h regime 降级为 `ranging` 且浮亏，V6.6 会用 `v66_trend_invalidated_by_range` 更快退出。
- 震荡止盈：震荡单回到箱体中部且已有小利润时，用 `v66_ranging_midbox_take_profit` 优先收掉，不幻想吃完整段。

V6.6 的仓位也比 V6.5 更克制：首笔 2,500 USDT、单次加仓 1,000 USDT、最大总仓位 4,500 USDT、账户最大亏损预算 2.5%，并且只允许一次加仓。

## 当前观察指标

优先看这些指标，不只看单笔盈亏：

- `profit_all_coin`：总收益。
- `trade_count` / `closed_trade_count`：交易频率与样本数量。
- 当前开仓数、持仓时长和单仓占用资金。
- 最大回撤与连续亏损。
- V6.3/V6.5 在同一行情下是否同时开仓、是否方向一致。
- V6.5 是否只是交易次数变多，还是扣除成本后收益/回撤比也改善。

## 下一步优化方向

短期先让 V6.3、V6.5 和 V6.6 跑出样本，不要同时改太多参数。

1. 先比较 V6.5 的交易频率是否真的带来收益/回撤改善。
2. 再比较 V6.6 是否用更低交易频率换来更好的收益/回撤比。
3. 再看 V6.6 的箱体边缘阈值是否过窄或过宽，必要时微调区间位置、RSI、布林位置和超时时间。
4. 再考虑资金费率过滤、交易时段过滤和成本视角评分。
5. 最后再扩展币种或引入新版本。

## 风险提示

当前仍处于 dry-run 和策略对比阶段。真实交易前，需要额外验证交易所最小下单量、资金费率、手续费、滑点、API 稳定性和极端行情下的止损行为。
