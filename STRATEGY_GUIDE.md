# 策略说明书

## 当前目标

当前线上不是追求“版本越多越好”，而是做一组干净的双 bot 对比：

- **V6.3**：当前稳定基线，在 V6.2 信号基础上按账户最大亏损预算决定加仓额度。
- **V6.4**：当前进攻挑战者，沿用 V6.3 的安全检查，但提高首仓、加仓额度、账户风险预算和波动容忍度。

我们要观察的是：V6.4 从 1h 改为 15m 主周期后，能否在震荡期捕捉更多机会，同时仍然受 4h 趋势过滤和账户风险预算约束。

## 核心机制

两个当前版本都基于 4h 判断市场状态。V6.3 仍在 1h 上找入场，V6.4 改为在 15m 上找入场。

- 趋势判断：EMA、ADX、DI、ATR 等指标确认方向和趋势强度。
- 入场确认：在趋势方向上等回调、反弹或突破。
- 出场控制：ROI、固定止损、自定义价格滑点和超时保护共同约束。
- 风险控制：每个 bot 独立 dry-run 数据库，当前采用单 BTC 仓、`max_open_trades=1`。

## V6.3 与 V6.4 的区别

| 项目 | V6.3 稳定基线 | V6.4 进攻挑战者 |
| --- | --- | --- |
| 趋势入场 | 保留 | 保留 |
| 震荡入场 | 关闭 | 关闭 |
| 保护机制 | Cooldown、StoplossGuard、MaxDrawdown | Cooldown、StoplossGuard、MaxDrawdown |
| 主交易周期 | 1h | 15m |
| 首笔仓位 | 1,500 USDT | 2,500 USDT |
| 单次加仓目标 | 1,000 USDT | 1,500 USDT |
| 最大总仓位 | 3,500 USDT | 5,000 USDT |
| 加仓盈利门槛 | 每次 1.2% 起 | 每次 0.8% 起 |
| 加仓冷却 | 60 分钟 | 30 分钟 |
| 账户最大亏损预算 | 1.5% | 2.5% |
| ATR 波动上限 | 3.0% | 4.0% |
| 额外门槛 | 止损/强平距离、波动、冷却、可用余额 | 同 V6.3，但阈值更进攻 |

V6.4 不是“把钱用满”的版本。它仍然按账户最大亏损预算反推可加仓额度：如果止损距离太近、强平距离太近、ATR 波动过高、冷却时间未到、可用余额不足，或者账户风险预算已经用完，都不会加仓。

## 当前观察指标

优先看这些指标，不只看单笔盈亏：

- `profit_all_coin`：总收益。
- `trade_count` / `closed_trade_count`：交易频率与样本数量。
- 当前开仓数、持仓时长和单仓占用资金。
- 最大回撤与连续亏损。
- V6.3/V6.4 在同一行情下是否同时开仓、是否方向一致。
- V6.4 是否只是收益变大，还是回撤、波动和补仓风险也同步变大。

## 下一步优化方向

短期先让 V6.3 和 V6.4 跑出样本，不要同时改太多参数。

1. 先比较 V6.4 的进攻仓位是否改善收益/回撤比。
2. 再看 V6.4 的加仓是否过早、过密，必要时回调 `min_scale_in_profit` 和 `min_scale_in_minutes`。
3. 再考虑资金费率过滤、交易时段过滤和成本视角评分。
4. 最后再扩展币种或引入新版本。

## 风险提示

当前仍处于 dry-run 和策略对比阶段。真实交易前，需要额外验证交易所最小下单量、资金费率、手续费、滑点、API 稳定性和极端行情下的止损行为。
